Why This Matters

The real cost of getting location decisions wrong

  • Store expansion is expensive — a bad location locks in rent, staffing, and inventory losses for years.
  • Decision-makers waste hours switching between Google Maps, spreadsheets, and ad-hoc SQL queries with no unified view.
  • There is no easy way to combine competitor density, foot traffic, and area demand signals into one coherent decision.
  • An agent that reasons over live tools removes this friction and turns research into action in seconds.
3 / 12

Solution Overview

What the agent does in one conversation

  • Accepts natural-language business questions: "Where should I open next?" or "How crowded is this area?"
  • Searches competitors and validates candidate locations through Google Maps MCP — place search, details, and distance queries.
  • Uses BigQuery MCP to discover available tables, write SQL automatically, and score demand signals relevant to the question.
  • Synthesises all findings into a ranked recommendation with clear reasoning and a direct Google Maps link.

Execution Flow

From business question to map-grounded recommendation

1
User types a business question in the ADK Web UI — no SQL, no APIs, just natural language.
2
root_agent (Gemini 2.5 Pro) plans the task and decides which tools to call — Maps MCP, BigQuery MCP, or both.
3
Maps MCP checks competitor density, place details, and validates candidate locations with live data.
4
BigQuery MCP lists available tables, auto-generates SQL, and returns demand and traffic signals.
5
Gemini synthesises all signals into a ranked recommendation with rationale and a Google Maps link.

Engineering Journey

Key challenges solved during development

  • Module recovery: agent.py was missing from disk — reconstructed the full source from .pyc bytecode cache constants and names.
  • Session 403 fix: Diagnosed silent startup failures by tracing MCP toolset init, ADC credential flow, and ADK session routing.
  • MCP stability: Stabilised Maps MCP and BigQuery MCP connections so the web app reliably creates sessions and returns responses.
  • Auth hardening: Validated MAPS_API_KEY env binding and Application Default Credentials for BigQuery bearer token generation.
