AWS in Plain English

The 4 services our agent audits

🔥Security Groups

Like a firewall for your cloud servers. Port 22 = SSH access. Port 3389 = Windows access. 0.0.0.0/0 = anyone on Earth can enter.

🔑IAM Policies

Controls WHO can do WHAT in your AWS account. action:* = this user can do absolutely everything — delete databases, steal data.

🪣S3 Buckets

Cloud storage like Google Drive. If set to public, anyone can download your files. Needs encryption (KMS/SSE) and versioning enabled.

🌐VPC & Logging

Your private network in AWS. VPC Flow Logs = recording traffic. CloudTrail = recording all actions. GuardDuty = threat detector. All must be ON.

Architecture

How the agent works — 5 steps

1
Environment sends a misconfigured AWS config
The benchmark server (FastAPI/Uvicorn) sends a JSON snapshot of a real AWS resource
2
Agent calls Qwen 72B via Hugging Face
inference.py sends the config + a detailed system prompt to the LLM using an OpenAI-compatible API
3
LLM returns structured JSON findings
findings, severity (HIGH/MEDIUM/LOW), recommendations, and a config_patch to fix each issue
4
Environment scores the response (0.0 → 1.0)
Benchmark checks if the agent found the right issues with the right AWS terminology
5
Agent retries with feedback (up to 5 steps)
If score is low, the agent re-analyses the full config using the feedback to improve recall

The Brain

Qwen 2.5-72B — Why this model?

72 Billion
Parameters — large enough to reason about complex cloud security configurations
Free API
Available via Hugging Face Router — no API cost, OpenAI-compatible endpoint

Key design decisions:

  • →JSON mode (response_format: json_object) forces valid structured output — no broken responses
  • →Temperature 0.2 — low randomness for consistent, reliable security findings every run
  • →2500 max tokens — enough room for 6–10 detailed findings with recommendations

The Secret Sauce

Prompt Engineering

// System prompt forces exact AWS benchmark terminology
"findings": ["Security group allows port 22 SSH from 0.0.0.0/0"]
"severity": ["HIGH"]
"recommendations": ["Restrict SSH to /32 CIDR, VPN or bastion"]
"config_patch": {"FlowLogsEnabled": true, "CloudTrailEnabled": true}
Exact Terms
Forces model to use SSH, RDP, 0.0.0.0/0, action:*, CloudTrail — exact words the benchmark scores on
Min 6 Findings
System prompt requires at least 6 separate findings — never combining multiple issues into one
Smart Fallback
If LLM fails to parse, a context-aware fallback submits relevant findings based on config keywords

Learnings

Challenges & how we solved them

Problem
LLM returned broken JSON → only 1 finding → reward 0.01 every step
✓ Fixed with response_format: json_object + multi-layer parsing fallback
Problem
Model used vague language → benchmark couldn't match keywords → low score
✓ Fixed by forcing exact AWS terms: “port 22 SSH”, “action:*”, “0.0.0.0/0” in system prompt
Problem
Agent gave up after 1 weak response with no improvement loop
✓ Fixed by building a 5-step retry loop with score-based feedback on each iteration
