🎯 Situation

The Problem

Data analysts and engineers spend significant time just to answer simple questions from S3 data.

❌ Without This Agent

  • Know the exact table schema
  • Write SQL with correct column names
  • Run in Athena or a notebook
  • Format and interpret the result
  • Handle format differences (CSV vs Parquet)
  • Log in to S3 console to upload data
  • 10–15 minutes minimum per question

✅ With This Agent

  • Type the question in plain English
  • Agent picks the right tool automatically
  • Schema, routing, SQL all handled internally
  • Upload datasets from the browser
  • Delete datasets from the browser
  • Re-run questions from query history
  • Answer in seconds, not minutes

🆕 Latest Changes

What are the additions

All previous features are preserved. These are the additions that make the system fully self-contained from the browser.

📂 Browser Upload

Upload CSV, Parquet, or JSON from the app. No S3 console. No server restart. Registers to Athena instantly.

🗑️ Browser Delete

Delete datasets from the UI. Removes from S3 and agent registry in one click. No manual cleanup needed.

🕘 Query History

All questions logged with timestamps in the sidebar. Click any past question to re-populate and re-run instantly.

🧠 Session Memory

Each browser tab gets a unique session ID via localStorage. Agent remembers context within each session.

🔐 IAM Hardening

Tighter IAM policy permissions added. Only necessary S3, Athena, Bedrock, and CloudWatch actions are allowed.

🔄 JSON Query Flow

The /query endpoint now returns clean JSON instead of SSE stream chunks. Eliminates JSON parse errors in browser.

🛡️ Guardrails Retained

Input and output Bedrock Guardrails remain in place. PII anonymization, content filters, and grounding checks intact.

📊 CloudWatch Retained

Every query, response, and guardrail event continues to be logged. 90-day retention, archived to S3.

How to Use the UI — Step by Step

1

Upload a Dataset

  • Click the upload area in the sidebar
  • Pick a CSV, Parquet, or JSON file
  • Enter a short name (e.g. iris)
  • Click Upload & Register
  • Uploads to S3 and registers instantly
2

Ask a Question

  • Type any question in the chat input
  • Press Send or hit Enter
  • Agent picks the right tool automatically
  • Answer appears in the chat panel

Re-run from History

  • All questions appear in the left sidebar
  • Newest questions at the top
  • Click any item to re-populate input
  • Send again to re-run it
  • Timestamp shown next to each entry
4

Delete a Dataset

  • Type the dataset name to delete
  • Click Delete Dataset
  • Removed from S3 automatically
  • Removed from agent registry
  • No S3 console access needed

A production-style agentic system — not a prototype — that could handle real data questions safely, at scale, from any browser.

  • Keep memory scoped per browser tab
  • Harden IAM permissions for least-privilege access

The Agent — Tools & Routing

ToolWhen CalledWhat It Does
list_datasetsUser asks what data is availableLists all datasets including browser-uploaded files with row and column counts
get_schemaAlways — before any data questionReturns column names, types, and 3 sample rows. Works for CSV, Parquet, and JSON
pandas_queryDatasets < 10,000 rowsIn-memory analysis. Auto-detects file format by extension. Fast and lightweight
get_athena_table_infoBefore every SQL queryConfirms exact column names to prevent schema mismatch errors in Athena
athena_sql_queryAggregations · large data · TOP NCOUNT · SUM · GROUP BY · ORDER BY. Max 2 retries, then falls back to pandas

🔐 Security

Guardrails & IAM Hardening

🛡️ Bedrock Guardrails — Input

  • Blocks harmful, hateful, or violent prompts
  • Anonymizes PII (names, emails, phone numbers)
  • Blocks prompt injection attempts
  • Denies off-topic or irrelevant queries
  • Agent never sees unsafe input

🛡️ Bedrock Guardrails — Output

  • Validates final response before user sees it
  • Grounding threshold: 0.7
  • Relevance threshold: 0.7
  • strip_thinking() removes internal reasoning tags

🔐 IAM Policy Hardening NEW

Least-privilege IAM policy added. Only the exact permissions needed are granted.

  • S3: GetObject, PutObject, DeleteObject on the specific bucket only
  • Athena: StartQuery, GetResults, StopQuery on the specific workgroup
  • Bedrock: InvokeModel on Nova Lite and the specific Guardrail ARN only
  • CloudWatch: CreateLogGroup, PutLogEvents on specific log group
  • No wildcard actions or resources permitted

🚀 End-to-End Flow

Query Lifecycle — How One Question Works

1

User Types Question

"Which state has the most farmers markets?"

Browser sends POST /query with the question and session_id.

2

Input Guardrail

Bedrock scans for harmful content and PII. Clean → passed to agent. Blocked → safe rejection returned immediately.

3

Agent + Tool Calls

Nova Lite calls get_schema, then get_athena_table_info, then athena_sql_query with a COUNT + GROUP BY query.

4

Output Guardrail + Answer

Response verified against grounding and relevance thresholds. strip_thinking() removes internal tags. JSON returned to browser.

→ "California with 1,528 farmers markets."
